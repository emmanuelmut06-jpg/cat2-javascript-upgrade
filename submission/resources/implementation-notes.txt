IMPLEMENTATION NOTES – Elite Shoes CAT 2
1. LOOP-RENDERED DYNAMIC CONTENT
- Two arrays of objects created: featuredProducts and newArrivals,
  each holding shoe data (name, price, image).
- A reusable function, renderProducts(productArray, containerId), loops
  through each array using forEach() and builds a product card for
  every item using createElement() and appendChild().
- Each card's "Add to Cart" button gets its click handler attached via
  addEventListener() inside the loop (instead of inline onclick, which
  is not allowed).
- Two empty containers in the HTML (#featured-grid and
  #new-arrivals-grid) are targeted separately, so the same render
  function handles both the Featured Collection and New Arrivals
  sections without duplicating code.

2. DYNAMIC ADD & REMOVE ELEMENTS (WISHLIST)
- A wishlist section lets the user type a shoe name into an input and
  click "Add to Wishlist."
- On click, the value is read using .value and trimmed to prevent
  empty/blank entries.
- addWishlistItem() creates a new <li> using createElement(), builds a
  "Remove" button inside it, and appends the whole thing to the list
  using appendChild().
- Each Remove button has its own click listener that calls
  li.remove(), deleting that specific wishlist item from the DOM when
  clicked.

3. FORM HANDLING WITH VALIDATION FEEDBACK
- The Contact section includes a real form (name, email, message)
  instead of static text.
- On submit, event.preventDefault() stops the default page reload so
  JS can handle validation itself.
- Each field's value is read using .value and trimmed to check for
  empty input.
- If any field is empty, or the email doesn't look valid (missing "@"
  or "."), an error message is written into #form-feedback using
  textContent, styled red.
- If all checks pass, a personalized success message is shown in
  green, and the form is cleared using contactForm.reset().

4. PERSISTENT STATE WITH LOCALSTORAGE
- The wishlist feature persists across page reloads using
  localStorage.
- On page load, JSON.parse(localStorage.getItem("wishlist")) retrieves
  any previously saved items (or starts as an empty array if none
  exist yet).
- Every time an item is added, it's pushed into the savedWishlist
  array and saved with localStorage.setItem(), using
  JSON.stringify() to convert the array into a storable string.
- When an item is removed, removeFromStorage() finds it in the array
  with indexOf(), removes it with splice(), and re-saves the updated
  array — so removed items stay gone after a refresh too.

5. CLICK-TO-REVEAL ON BANNER
- Clicking the hero banner (#hero-banner) toggles a "show-caption"
  class on it using classList.toggle().
- CSS hides a caption (.hidden-caption) by default, and only displays
  it when the parent .hero has the "show-caption" class, using a
  compound selector.
- This creates a click-to-reveal effect on the banner without changing
  its layout when the caption is hidden.
